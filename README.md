# Forecast — Прогноз расхода запасных частей

Streamlit-приложение для прогнозирования спроса на запасные части подъёмной техники. Загружает два Excel-отчёта из 1С, группирует запчасти по аналогам и строит прогноз продаж и ремонта на срок до 12 месяцев.

## Как это работает

1. Загружаются два отчёта: «Запчасти списанные в ремонт» и «Остатки и обороты»
2. Запчасти нормализуются и группируются по аналогам через граф артикулов
3. Для каждой группы строится временной ряд продаж и ремонта
4. Автоматически выбирается лучший метод прогноза:
   - **TSB** — для прерывистого спроса (много нулей)
   - **Holt-Winters** — при наличии сезонности (≥ 24 месяца истории)
   - **Holt / ETS** — для коротких рядов
   - **Smoothed-ETS / Smoothed-Holt / Smoothed-HW** — если модель сглаживания выбрана корректно, но прогноз получился слишком заниженным
   - **Recent-median-decay / Recent-nonzero-decay** — запасные варианты, если модель сглаживания схлопнулась в ноль
   - **Mean-6m** — резервный вариант

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
streamlit run app/main.py
```

## Входные данные

Два Excel-отчёта из 1С (`.xlsx` или `.xls`):

| Файл | Описание |
|------|----------|
| Запчасти списанные в ремонт | Расход запчастей на ремонт техники |
| Остатки и обороты | Складские обороты: приход, расход, остаток |

Формат файлов определяется автоматически — служебные строки перед заголовками пропускаются.

## Конфигурация

Ручные правки хранятся в `configs/`:

- `corrections_repair.json` — правки артикулов и номенклатуры для отчёта по ремонту
- `corrections_stock.json` — правки для складского отчёта
- `matches.json` — матчинг составных запчастей (комплектов) на отдельные артикулы

## Параметры прогноза

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| Горизонт | 3 мес. | Количество месяцев вперёд (1–12) |
| IQR × | 1.5 | Чувствительность очистки выбросов |
| Порог нулей для TSB | 0.50 | Доля нулей выше которой применяется TSB |
| Показать очищенный ряд | Вкл | Отображать ряд после замены выбросов |
| Учитывать текущий месяц | Вкл | Включать незавершённый текущий месяц в обучение |

## Логирование

Действия пользователей логируются в Google Sheets. Для настройки создай `.streamlit/secrets.toml`:

```toml
google_sheet_id = "id_таблицы"

[gcp_service_account]
type = "service_account"
project_id = "..."
private_key_id = "..."
private_key = "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
client_email = "...@....iam.gserviceaccount.com"
client_id = "..."
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
auth_provider_x509_cert_url = "https://www.googleapis.com/oauth2/v1/certs"
client_x509_cert_url = "..."
```

Файл `secrets.toml` используется только локально, добавлен в `.gitignore` и не должен попадать в репозиторий.

Логируются следующие события:
- Загрузка и обработка файлов
- Построение прогноза по каждой группе
- Ошибки
